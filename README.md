# Αλγόριθμοι Συνδυαστικής Βελτιστοποίησης για Προβλήματα Χωροθέτησης Θυρίδων Παράδοσης Δεμάτων

Θέμα: Αλγόριθμοι Συνδυαστικής Βελτιστοποίησης για Προβλήματα Χωροθέτησης Θυρίδων Παράδοσης Δεμάτων<br>
Συγγραφέας: Ειρήνη Μαρκαντώνη<br>
Επιβλέπων Καθηγητής: Νικόλαος Πλόσκας<br>
Τμήμα: Ηλεκτρολόγων Μηχανικών και Μηχανικών Υπολογιστών<br>
Ίδρυμα: Πανεπιστήμιο Δυτικής Μακεδονίας<br>
Ημερομηνία: Φεβρουάριος 2026<br>

Το παρόν αποθετήριο περιλαμβάνει κώδικα που αναπτύχθηκε στο πλαίσιο της διπλωματικής εργασίας με θέμα: «Αλγόριθμοι Συνδυαστικής Βελτιστοποίησης για Προβλήματα Χωροθέτησης Θυρίδων Παράδοσης Δεμάτων». Στόχος της εργασίας είναι ο προσδιορισμός των βέλτιστων θέσεων και τύπων θυρίδων (parcel lockers) σε αστικό περιβάλλον, με σκοπό την ελαχιστοποίηση της συνολικής απόστασης που διανύουν οι πελάτες για την παραλαβή των δεμάτων τους.
Το πρόβλημα διατυπώνεται ως ένα ετερογενές πρόβλημα χωροθέτησης εγκαταστάσεων — επέκταση του p-median με περιορισμούς χωρητικότητας, κόστους, προσβασιμότητας και περιβαλλοντικής επίδρασης. Η μοντελοποίηση λαμβάνει υπόψη τρεις τύπους θυρίδων (Small / Medium / Large), πέντε μέσα μεταφοράς με διαφορετικά όρια προσβασιμότητας και στοχαστική κατανομή ζήτησης με χωρική ετερογένεια, ώστε να προσεγγίζει ρεαλιστικά σενάρια αστικής εφοδιαστικής.
Η επίλυση βασίστηκε σε Μικτό Ακέραιο Γραμμικό Προγραμματισμό (MILP) με τον λύτη Gurobi, καθώς και στην ανάπτυξη custom αλγορίθμων Branch & Bound σε δύο εκδόσεις — μια baseline και μια βελτιωμένη που ενσωματώνει ευρετικές και μεθευρετικές τεχνικές (VND, Simulated Annealing, ILS) για την αντιμετώπιση της υπολογιστικής πολυπλοκότητας. 

Το αποθετήριο περιλαμβάνει:

- Γεννήτρια συνθετικών προβλημάτων για instances διαφορετικού μεγέθους και πολυπλοκότητας
- Μοντέλα MILP υλοποιημένα με Gurobi
- Custom υλοποιήσεις αλγορίθμων Branch & Bound (baseline και βελτιωμένη έκδοση)

Ο παρών κώδικας αναπτύχθηκε για τη δημιουργία συνθετικών συνόλων δεδομένων (test instances) για το πρόβλημα χωροθέτησης ετερογενών εγκαταστάσεων, με εφαρμογή σε συστήματα θυρίδων παραλαβής δεμάτων (parcel lockers). Η γεννήτρια στοχεύει στην προσομοίωση ρεαλιστικών αστικών συνθηκών, ενσωματώνοντας διαφορετικούς τύπους εγκαταστάσεων, ποικιλία ζήτησης πελατών, περιορισμούς προσβασιμότητας και οικονομικά κριτήρια. Τα παραγόμενα instances μπορούν να χρησιμοποιηθούν για την αξιολόγηση και σύγκριση αλγορίθμων βελτιστοποίησης, όπως μοντέλα Μεικτού Ακέραιου Γραμμικού Προγραμματισμού (MILP) ή custom μεθόδους (π.χ. Branch and Bound).


## Βασικά χαρακτηριστικά

### 1. Ετερογενείς εγκαταστάσεις
- Τρεις τύποι θυρίδων:

  - Large (0.1633 m³)

  - Πόδια

  - ΜΜΜ

1. Βασικές παραμέτρους (grid_size, clients, facilities, p)

##  Μοντέλο Βελτιστοποίησης (`model.py`)

Ο παρών κώδικας υλοποιεί ένα **Mixed Integer Linear Programming (MILP)** μοντέλο για το πρόβλημα χωροθέτησης ετερογενών θυρίδων παράδοσης δεμάτων (*parcel lockers*). Το μοντέλο αποτελεί επέκταση του προβλήματος **capacitated p-median**, καθώς λαμβάνει υπόψη διαφορετικούς τύπους εγκαταστάσεων, περιορισμούς χωρητικότητας, ζήτησης, προσβασιμότητας και προϋπολογισμού.

Στόχος του μοντέλου είναι η επιλογή των βέλτιστων θέσεων και τύπων εγκαταστάσεων, έτσι ώστε να ελαχιστοποιείται η συνολική απόσταση εξυπηρέτησης των πελατών από τις επιλεγμένες θυρίδες.

Το μοντέλο επιλύεται με τον **Gurobi Optimizer** και υποστηρίζει τόσο την επίλυση μεμονωμένων προβλημάτων όσο και τη μαζική επίλυση πολλαπλών instances από έναν φάκελο.



###  Το μοντέλο περιλαμβάνει:

- **Μεταβλητές απόφασης**
  - `x[i,j]`: παίρνει τιμή 1 αν ανοίγει εγκατάσταση τύπου `j` στη θέση `i`
  - `y[k,i]`: παίρνει τιμή 1 αν ο πελάτης `k` εξυπηρετείται από τη θέση `i`

- **Αντικειμενική συνάρτηση**
  - Ελαχιστοποίηση της συνολικής απόστασης πελατών–εγκαταστάσεων

- **Βασικούς περιορισμούς**
  - Επιλογή ακριβώς `P` εγκαταστάσεων
  - Το πολύ ένας τύπος εγκατάστασης ανά θέση
  - Επιτρεπτοί τύποι εγκαταστάσεων ανάλογα με τη διαθέσιμη χωρητικότητα/όγκο της θέσης
  - Ανάθεση κάθε πελάτη σε ακριβώς μία εγκατάσταση
  - Εξυπηρέτηση πελατών μόνο από ανοιχτές εγκαταστάσεις
  - Περιορισμοί προσβασιμότητας ανά μέσο μεταφοράς
  - Περιορισμοί χωρητικότητας εγκαταστάσεων
  - Περιορισμός συνολικού προϋπολογισμού



## Χαρακτηριστικά μοντέλου

- **Ετερογενείς εγκαταστάσεις:**
  - Small, Medium, Large θυρίδες  
  - Διαφορετικά κόστη και χωρητικότητες ανά τύπο

- **Ρεαλιστική προσβασιμότητα:**
  - 5 μέσα μεταφοράς  
  - Διαφορετικά όρια απόστασης ανά μέσο

- **Οικονομική και περιβαλλοντική διάσταση**
  - Κόστος εγκατάστασης θυρίδων
  - Κόστος μετακίνησης ανά χιλιόμετρο
  - Εκπομπές CO₂ ανά μέσο μεταφοράς
  - Μετατροπή εκπομπών CO₂ σε οικονομικό κόστος

- **Συνδυαστικό κόστος:**
  - Κόστος εγκατάστασης + μετακίνησης + περιβαλλοντικό κόστος  

- **Benchmarking:**
  - Οι λύσεις του Gurobi χρησιμοποιούνται ως σημείο αναφοράς για σύγκριση με heuristics και Branch & Bound

##  Μοντέλο για Branch & Bound (`model_for_bnb.py`)

Ο παρών κώδικας υλοποιεί ένα **MILP μοντέλο ειδικά σχεδιασμένο για χρήση σε custom αλγορίθμους Branch & Bound**, χρησιμοποιώντας **μονοδιάστατο διάνυσμα μεταβλητών απόφασης** αντί για την κλασική δισδιάστατη αναπαράσταση.

Σε αντίθεση με το βασικό μοντέλο (`model.py`), εδώ όλες οι μεταβλητές (εγκαταστάσεων και αναθέσεων) αποθηκεύονται σε ένα ενιαίο vector, γεγονός που επιτρέπει άμεση ενσωμάτωση σε custom αλγορίθμους βελτιστοποίησης.

Το αρχείο λειτουργεί ως βοηθητικό module και παρέχει τη συνάρτηση `run_model(filename)`, η οποία διαβάζει ένα instance και επιστρέφει όλα τα απαραίτητα στοιχεία για την εκτέλεση Branch and Bound.



##  Σκοπός

Ο κώδικας χρησιμοποιείται για:

- Δημιουργία του LP relaxation του προβλήματος  
- Μονοδιάστατη αναπαράσταση μεταβλητών  
- Καθορισμό ορίων μεταβλητών  
- Καθορισμό ακέραιων μεταβλητών  
- Υποστήριξη custom Branch and Bound  


##  Δομή μονοδιάστατου διανύσματος

Το διάνυσμα μεταβλητών έχει μέγεθος:

```text
num_vars = N * E + CL * N + 1
```

**Δομή:**
```text
[ x[i,j] μεταβλητές | y[k,i] μεταβλητές | objective ]
```
**Αναλυτικά:**
- Θέσεις 0 έως N×E - 1: μεταβλητές εγκατάστασης x[i,j]
- Θέσεις N×E έως N×E + CL×N - 1: μεταβλητές ανάθεσης y[k,i]
- Τελευταία θέση: αντικειμενική συνάρτηση

**Αντιστοίχιση:**
```text
x[i,j] → i * E + j
y[k,i] → N * E + k * N + i
objective → N * E + CL * N
```

##  Σχέση με το βασικό μοντέλο (`model.py`)

Οι περιορισμοί και η αντικειμενική συνάρτηση του μοντέλου είναι ίδιοι με αυτούς που περιγράφονται στο αρχείο `model.py`.

Η βασική διαφορά του παρόντος αρχείου (`model_for_bnb.py`) έγκειται αποκλειστικά στη **μορφή αναπαράστασης των μεταβλητών απόφασης**, η οποία είναι μονοδιάστατη, ώστε να είναι συμβατή με custom υλοποιήσεις αλγορίθμων Branch and Bound.

Συγκεκριμένα:

- Η μαθηματική διατύπωση του προβλήματος παραμένει **αμετάβλητη**
- Όλοι οι περιορισμοί (χωρητικότητας, προσβασιμότητας, ανάθεσης, προϋπολογισμού κ.λπ.) είναι **ίδιοι**
- Η αντικειμενική συνάρτηση είναι επίσης **ίδια** (ελαχιστοποίηση συνολικής απόστασης)

Η μόνη διαφοροποίηση είναι η αντιστοίχιση των μεταβλητών σε ένα ενιαίο διάνυσμα, αντί για δισδιάστατες δομές.


##  Είσοδος

Η συνάρτηση:

```python
run_model(filename)
```
δέχεται ως είσοδο ένα αρχείο .txt που έχει παραχθεί από τον generator (flpgenerator_BoxNow.py)


## Έξοδος
Η συνάρτηση run_model(filename) δεν επιστρέφει μόνο το μοντέλο, αλλά ένα σύνολο δομών που χρησιμοποιούνται από τον Branch and Bound:

```python
model, ub, lb, integer_var, num_vars, N, E, CL, P, cl_sp_distances, transport_mode, emission_values, cost_types, facility_types, Cmax, capacities, demand, get_max_distance, combined_cost_per_km
```
**Περιγραφή εξόδου:**
- model: Το Gurobi μοντέλο (LP relaxation του προβλήματος)
- ub: Λίστα με τα άνω όρια των μεταβλητών
- lb: Λίστα με τα κάτω όρια των μεταβλητών
- integer_var: Boolean λίστα που δηλώνει ποιες μεταβλητές πρέπει να είναι ακέραιες (για B&B)
- num_vars: Συνολικός αριθμός μεταβλητών στο μονοδιάστατο διάνυσμα
- N: Αριθμός υποψήφιων εγκαταστάσεων
- E: Αριθμός τύπων εγκαταστάσεων
- CL: Αριθμός πελατών
- P: Αριθμός εγκαταστάσεων που πρέπει να ανοίξουν
- cl_sp_distances: Πίνακας αποστάσεων πελάτη–εγκατάστασης
- transport_mode: Μέσο μεταφοράς κάθε πελάτη
- emission_values: Εκπομπές CO₂ ανά μέσο μεταφοράς
- cost_types: Κόστος εγκατάστασης ανά τύπο θυρίδας
- facility_types: Μέγιστος επιτρεπτός τύπος εγκατάστασης ανά θέση
- Cmax: Διαθέσιμο κεφάλαιο
- capacities: Χωρητικότητες εγκαταστάσεων
- demand: Ζήτηση πελατών
- get_max_distance: Συνάρτηση που επιστρέφει τη μέγιστη επιτρεπτή απόσταση ανά μέσο μεταφοράς
- combined_cost_per_km: Συνδυασμένο κόστος (οικονομικό + περιβαλλοντικό) ανά km

Το model_for_bnb.py δεν εκτελεί πλήρη επίλυση του προβλήματος.

## Baseline Branch & Bound (`simple_bnb.py`)
Ο παρών κώδικας υλοποιεί τη βασική έκδοση του αλγορίθμου **Branch and Bound** για την επίλυση του ετερογενούς προβλήματος χωροθέτησης θυρίδων παράδοσης δεμάτων (*parcel lockers*).

Ο αλγόριθμος χρησιμοποιεί το μοντέλο που δημιουργείται στο αρχείο `model_for_bnb.py`, όπου όλες οι μεταβλητές απόφασης έχουν αναπαρασταθεί σε ένα μονοδιάστατο διάνυσμα. Η μορφή αυτή διευκολύνει τη διαχείριση των ορίων των μεταβλητών κατά τη διαδικασία του branching.

Η συγκεκριμένη υλοποίηση λειτουργεί ως **baseline αλγόριθμος**, δηλαδή ως βασική μέθοδος αναφοράς για σύγκριση με βελτιωμένες εκδόσεις Branch and Bound ή άλλες ευρετικές/μεθευρετικές προσεγγίσεις.


##  Σκοπός

Ο σκοπός του αρχείου είναι:

- η επίλυση MILP προβλημάτων μέσω custom Branch and Bound,
- η επίλυση LP relaxations σε κάθε κόμβο,
- η εύρεση ακέραιων εφικτών λύσεων,
- η παρακολούθηση άνω και κάτω φραγμάτων,
- η καταγραφή χρόνου εκτέλεσης και αριθμού εξερευνημένων κόμβων,
- η παραγωγή αποτελεσμάτων για πειραματική σύγκριση.



##  Βασικά χαρακτηριστικά

Ο αλγόριθμος περιλαμβάνει:

- **Depth-First Search (DFS)**  
  Η εξερεύνηση του δέντρου γίνεται με στοίβα (`stack`), δηλαδή ακολουθείται στρατηγική LIFO.

- **First Fractional Variable**  
  Για branching επιλέγεται η πρώτη μεταβλητή που έχει κλασματική τιμή στην LP λύση.

- **Branching σε δύο κόμβους**
  - Αριστερός κόμβος:  
    `x_i ≤ floor(x_i)`
  - Δεξιός κόμβος:  
    `x_i ≥ ceil(x_i)`

- **Bounding**
  Κόμβοι απορρίπτονται όταν το LP relaxation δεν μπορεί να οδηγήσει σε καλύτερη ακέραια λύση.

- **Warm start**
  Χρησιμοποιούνται οι simplex βάσεις (`VBasis`, `CBasis`) από τον γονικό κόμβο, ώστε να επιταχύνεται η επίλυση των LP subproblems.

- **Χρονικό όριο**
  Ο αλγόριθμος σταματά μετά από `3600` δευτερόλεπτα, δηλαδή 1 ώρα.








## Λογική λειτουργίας

Ο αλγόριθμος εκτελεί τα εξής βήματα:

1. Επίλυση της **χαλάρωσης (LP relaxation)** του προβλήματος  
2. Έλεγχος αν η λύση είναι ακέραια  
3. Αν όχι:
   - Επιλογή κλασματικής μεταβλητής  
   - Δημιουργία δύο υποπροβλημάτων (branching)  
4. Εξερεύνηση κόμβων με **DFS**  
5. Κλάδεμα κόμβων όταν:
   - το πρόβλημα είναι μη εφικτό  
   - δεν μπορεί να βελτιώσει τη βέλτιστη λύση  
6. Ενημέρωση των **bounds** μέχρι σύγκλιση  



##  Δομή κόμβου (Node)

Κάθε κόμβος του δέντρου αποθηκεύει:

- Όρια μεταβλητών (LB / UB)  
- Βάθος στο δέντρο  
- Βάσεις simplex (warm start)  
- Μεταβλητή διακλάδωσης  
- Ετικέτα (root, left, right)

##  Βελτιωμένος Branch & Bound (`improved_bnb.py`)

Ο παρών κώδικας υλοποιεί μια **βελτιωμένη έκδοση του αλγορίθμου Branch and Bound (B&B)** για το ετερογενές πρόβλημα χωροθέτησης θυρίδων παράδοσης δεμάτων (*Heterogeneous Facility Location Problem – HFLP*).

Σε αντίθεση με την απλή έκδοση, η συγκεκριμένη προσέγγιση ενσωματώνει:

- ευρετικούς αλγορίθμους (heuristics)
- μεθευρετικές μεθόδους (metaheuristics)
- έξυπνη επιλογή μεταβλητής διακλάδωσης
- υβριδική στρατηγική εξερεύνησης
- τεχνικές επιτάχυνσης του B&B

 Στόχος είναι η **δραστική μείωση του χρόνου εκτέλεσης και του αριθμού κόμβων**, διατηρώντας υψηλή ποιότητα λύσεων, όπως επιβεβαιώνεται και πειραματικά στη διπλωματική εργασία.


### 1. Ευρετικοί Αλγόριθμοι (Heuristics)
Οι ευρετικοί χρησιμοποιούνται για την εύρεση μιας αρχικής εφικτής λύσης (incumbent), η οποία χρησιμοποιείται ως άνω φράγμα (upper bound).

| Μέθοδος           | Περιγραφή                                      | Ρόλος                |
| ----------------- | ---------------------------------------------- | -------------------- |
| Distance Greedy   | Επιλογή θέσεων βάσει απόστασης                 | Γρήγορη κάλυψη       |
| Cost-Aware Greedy | Ελαχιστοποίηση κόστους                         | Οικονομική λύση      |
| Feasibility-First | Προτεραιότητα στη χωρητικότητα                 | Εγγυημένη εφικτότητα |
| GRASP Multistart  | Τυχαιοποιημένο greedy με πολλαπλές επαναλήψεις | Διαφορετικές λύσεις  |


 Οφέλη:

- ταχεία εύρεση λύσης
- σημαντική μείωση search space

### 2. Μεθευρετικές Μέθοδοι (Metaheuristics)
Μετά τους heuristics, εφαρμόζονται metaheuristics για βελτίωση της λύσης.

| Μέθοδος               | Περιγραφή                             | Πλεονέκτημα             |
| --------------------- | ------------------------------------- | ----------------------- |
| VND                   | Διαδοχική αλλαγή neighborhoods        | Ισχυρή τοπική αναζήτηση |
| Simulated Annealing   | Πιθανότητα αποδοχής χειρότερων λύσεων | Escape από local minima |
| Iterated Local Search | Επαναληπτική τοπική αναζήτηση         | Εξερεύνηση λύσεων       |



Ρόλος:

- βελτιώνουν το incumbent
- δίνουν καλύτερο upper bound

### 3. Μετρικές (Metrics)
Χρησιμοποιούνται για branching, pruning και αξιολόγηση.

**Μετρικές πελατών**:
  
| Metric            | Περιγραφή                     |
| ----------------- | ----------------------------- |
| Customer Scarcity | Πόσες επιλογές έχει ο πελάτης |
| Demand            | Ζήτηση πελάτη                 |

**Μετρικές θέσεων**:

| Metric           | Περιγραφή                    |
| ---------------- | ---------------------------- |
| Feasible Count   | Πόσοι πελάτες εξυπηρετούνται |
| Feasible Demand  | Συνολική ζήτηση              |
| Inverse Distance | Κοντινή ζήτηση               |
| Site Scarcity | Πόσο κρίσιμη είναι μια θέση για την κάλυψη πελατών με περιορισμένες επιλογές |


**Capacity Pressure**:<br>
pressure = demand - capacity

 Χρησιμοποιείται για:

- επιλογή εγκαταστάσεων
- prioritization στο branching

### 4. Intelligent Branching
Η επιλογή μεταβλητής διακλάδωσης γίνεται με υβριδικό κριτήριο:

| Κριτήριο          | Ρόλος                     |
| ----------------- | ------------------------- |
| Fractionality     | Πόσο “μη ακέραιη” είναι   |
| Capacity pressure | Έλλειμμα χωρητικότητας    |
| Customer scarcity | Πελάτες με λίγες επιλογές |
| Site importance   | Σημαντικότητα θέσης (πόσο κρίσιμη είναι μια θέση για την κάλυψη πελατών με περιορισμένες επιλογές)      |


- Προτεραιότητα μεταβλητών
   - x[i,j] (άνοιγμα εγκαταστάσεων)
   - y[k,i] (αναθέσεις πελατών)

- Φάσεις αναζήτησης

| Φάση | Περιγραφή            |
| ---- | -------------------- |
| Dive | Γρήγορη εύρεση λύσης |
| Best | Βελτιστοποίηση       |




### 5. Βελτιώσεις Branch and Bound

| Τεχνική                | Περιγραφή                  |
| ---------------------- | -------------------------- |
| DFS Diving             | Γρήγορο εύρεση incumbent   |
| Best-First             | Βελτίωση λύσης             |
| Warm Start             | Επαναχρησιμοποίηση simplex |
| Constraint Propagation | Μείωση search space        |
| Deferred Nodes         | Αποθήκευση κόμβων          |



 Οφέλη:

- λιγότεροι κόμβοι
- ταχύτερη σύγκλιση
